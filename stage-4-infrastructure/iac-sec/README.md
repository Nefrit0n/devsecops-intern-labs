## Зачем сканировать IaC

Infrastructure as Code — это Dockerfile, Kubernetes YAML, Terraform HCL, Ansible playbooks. Это *чертежи* инфраструктуры. Ошибка в чертеже → дыра в production.

Классический пример: `ingress { cidr_blocks = ["0.0.0.0/0"] }` в Terraform. Одна строка — весь интернет получает доступ к вашей базе данных. IaC-сканер поймает это *до деплоя*.

**Связь с модулем 4.1:** там вы сканировали *собранный* образ. Здесь вы сканируете *исходники* инфраструктуры — Dockerfile, K8s манифесты, Helm charts. Ловите проблемы на этапе написания, а не после сборки.
