# Модуль 5.1 · CI/CD pipeline — автоматизация безопасности

**Время:** ~4 часа · **Тип:** 🟢 практика
**Привязка к ГОСТу:** 5.2 (инструментальная среда), 5.16 (безопасность при выпуске)

---

## Три уровня автоматизации

| Уровень | Когда срабатывает | Скорость | Глубина | Что запускаем |
|---------|-------------------|----------|---------|---------------|
| **Pre-commit** | До коммита (локально) | Секунды | Мелко | Gitleaks, hadolint, Semgrep (quick) |
| **PR pipeline** | На каждый Pull Request | 2–5 мин | Средне | SAST, SCA, IaC scan |
| **Main pipeline** | При мерже в main | 15–30 мин | Глубоко | Build, container scan, DAST, SBOM, sign |

**Принцип:** чем раньше — тем быстрее, но мельче. Чем позже — тем глубже, но медленнее. Нельзя запускать ZAP full scan на каждый коммит (30 мин), но можно на nightly.
