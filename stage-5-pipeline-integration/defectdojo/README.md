## Зачем нужен DefectDojo

На этапах 1–4 вы запускали ~30 инструментов. Каждый сгенерировал отчёт в своём формате. Без centralized management вы утонете:

- Semgrep нашёл 45 findings, Trivy — 120, ZAP — 67, Checkov — 33… **Итого 265.** Но сколько из них *уникальных*? (Подсказка: многие дублируются.)
- Одна и та же CVE может быть найдена Trivy (SCA), Grype (container), и npm audit. Это один баг или три?
- Кто отвечает за исправление? Какой SLA? Когда deadline?

DefectDojo решает всё это: импорт из 200+ инструментов, дедупликация, SLA tracking, дашборд, JIRA-интеграция.
