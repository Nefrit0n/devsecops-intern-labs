# Как контрибьютить

Спасибо за интерес к проекту! Вот как вы можете помочь.

## Что можно сделать

- Добавить задания или примеры к существующим этапам
- Наполнить этапы 1–5 (сейчас там заглушки)
- Улучшить документацию
- Добавить поддержку новых инструментов
- Расширить Juice Shop новыми уязвимостями
- Исправить ошибки

## Правила

1. Один PR = одна задача
2. Следуйте структуре существующих модулей (README.md + tasks.md + examples/ + templates/)
3. Все задания должны быть привязаны к конкретным разделам ГОСТ Р 56939-2024
4. Используйте только open-source инструменты
5. Пишите на русском (основной язык репозитория)

## Структура модуля

```
stage-N/module-name/
├── README.md      ← теория: о чём модуль, что читать, привязка к ГОСТу
├── tasks.md       ← задания + чеклист самопроверки
├── examples/      ← примеры конфигов, отчётов, скриптов
└── templates/     ← шаблоны для выполнения заданий
```
