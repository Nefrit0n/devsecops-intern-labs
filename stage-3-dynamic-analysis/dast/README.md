## Что такое DAST

DAST (**D**ynamic **A**pplication **S**ecurity **T**esting) — это тестирование *работающего* приложения. Вы не смотрите в код — вы отправляете HTTP-запросы и анализируете ответы. Чёрный ящик.

**Что DAST делает:**
1. **Crawling** — обходит приложение, собирает все URL, формы, параметры
2. **Passive scan** — анализирует ответы: заголовки, cookies, информационные утечки
3. **Active scan** — подставляет payloads (SQL injection, XSS, path traversal) и смотрит на реакцию

**Четыре инструмента — четыре подхода:**

| Инструмент | Метафора | Что делает лучше остальных |
|------------|----------|---------------------------|
| **ZAP** | «Швейцарский нож» | Всё: proxy, crawl, active scan, API, CI |
| **Nuclei** | «Снайпер с каталогом» | 11000 шаблонов на известные CVE/мисконфиги |
| **Nikto** | «Быстрый досмотр» | Серверные мисконфиги, дефолты, утечки |
| **Wapiti** | «Упрямый тестировщик» | Crawl + inject в каждый параметр |
