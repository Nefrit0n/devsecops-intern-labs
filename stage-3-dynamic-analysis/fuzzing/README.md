## Чем фаззинг отличается от DAST

| | DAST (ZAP, Nuclei) | Fuzzing (RESTler, ffuf) |
|--|---------------------|------------------------|
| Стратегия | Известные payloads из БД | Случайные / мутированные данные |
| Цель | Найти *известные* типы уязвимостей | Найти *неожиданное* поведение |
| Результат | «XSS в поле search» | «500 Internal Server Error при вводе 10MB строки» |
| Coverage | Широкий, но предсказуемый | Может найти что-то, о чём никто не думал |

**Аналогия:** DAST — это как проверять дверь ключами от известных замков. Fuzzing — это как бить по двери случайными предметами и смотреть, что сломается.

ГОСТ 5.13 (нефункциональное тестирование) включает фаззинг как один из методов. Это важно, потому что фаззинг находит уязвимости, которые DAST пропускает: крэши, memory leaks, unhandled exceptions, resource exhaustion.

---

## Три фаззера — три философии

| Инструмент | Философия | Что ищет |
|------------|-----------|----------|
| **RESTler** | «Я понимаю структуру твоего API» | 500-е ошибки, resource leaks, auth bypass через цепочки |
| **Schemathesis** | «Я проверяю контракт API» | Response не соответствует schema, неожиданные status codes |
| **ffuf** | «Я перебираю всё подряд» | Скрытые эндпоинты, параметры, backup-файлы |
