## Почему API-тестирование — отдельный модуль

ZAP и Nuclei тестируют *известные паттерны* уязвимостей (SQL injection, XSS). Но самые опасные API-уязвимости — это **проблемы бизнес-логики**, которые автоматические сканеры не понимают:

| OWASP API Top 10 | Описание | Автосканер найдёт? |
|-------------------|----------|--------------------|
| **API1: BOLA** | Доступ к чужим ресурсам по ID | ✗ (нужен контекст двух пользователей) |
| **API2: Broken Auth** | Слабые токены, отсутствие rate limit | Частично |
| **API3: Excessive Data** | API возвращает больше данных, чем нужно | ✗ (нужно знать что «лишнее») |
| **API4: Unrestricted Resource** | Нет лимита на создание ресурсов | ✗ |
| **API5: BFLA** | Вызов функций, на которые нет прав | ✗ (нужно знать роли) |

Для этого нужны **ручные тесты** (Postman), **контрактное тестирование** (Dredd), и **умные фаззеры** (CATS).

**crAPI** — идеальная мишень: мультисервисное API-приложение с уязвимостями из OWASP API Security Top 10.

---

## Три инструмента — три уровня

| Инструмент | Уровень | Что проверяет |
|------------|---------|---------------|
| **Postman + Newman** | Ручные тесты + CI | OWASP API Top 10, бизнес-логика |
| **Dredd** | Contract testing | API ведёт себя как описано в spec? |
| **CATS** | Авто-fuzzing из spec | 50+ fuzzers, injection, auth, schema |
