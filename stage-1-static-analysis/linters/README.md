## SAST ≠ Linter: в чём разница

| | SAST (Semgrep, Bandit) | Linter (ESLint, Ruff) |
|--|------------------------|----------------------|
| Цель | Найти уязвимости | Обеспечить единообразие и качество |
| Когда запускать | В CI/CD | В IDE + pre-commit + CI/CD |
| Пример finding | «SQL injection в строке 42» | «Используется `var` вместо `const`» |
| Security-фокус | Основной | Дополнительный (через плагины) |

Линтеры с security-плагинами — это **первая линия обороны**. Они работают в IDE, дают мгновенную обратную связь разработчику *пока он пишет код*. SAST запускается позже, в CI. Вместе они создают два рубежа.

**ГОСТ 5.8** требует не просто «используйте линтер», а **задокументированные правила кодирования** — coding standard, который команда принимает и которому следует.

---

## Три линтера — три слоя

| Инструмент | Что линтит | Зачем именно он |
|------------|-----------|-----------------|
| **ESLint + security** | JS/TS код | Стандарт индустрии. Security-плагины ловят DOM XSS, prototype pollution |
| **Ruff** | Python код | Сверхбыстрый. Встроены Bandit-правила как S-коды |
| **hadolint** | Dockerfile | Ловит `USER root`, `latest` tag, лишние пакеты — мост к этапу 4 |
