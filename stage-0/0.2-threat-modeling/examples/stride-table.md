# Пример STRIDE-таблицы

> Это пример для ориентира. Ваш анализ для Juice Shop будет отличаться.

| # | Компонент / поток      | S | T | R | I | D | E | Описание угрозы                          | Severity | Мера                    |
|---|------------------------|---|---|---|---|---|---|------------------------------------------|----------|-------------------------|
| 1 | User → Frontend (HTTP) | ✓ |   |   | ✓ |   |   | Подмена сессии; перехват данных без TLS   | HIGH     | HTTPS + Secure cookies  |
| 2 | Frontend → Backend API |   | ✓ |   |   |   | ✓ | CSRF; инъекция в тело запроса             | HIGH     | CSRF-токен + валидация  |
| 3 | Backend → Database     |   | ✓ |   | ✓ |   |   | SQL injection через user input            | HIGH     | Параметризованные запросы |
| 4 | Backend (логирование)  |   |   | ✓ |   |   |   | Отсутствие audit log                      | MEDIUM   | Structured logging      |
| 5 | Frontend (JS)          |   |   |   |   | ✓ |   | ReDoS через пользовательский ввод         | LOW      | Ограничение regex       |
