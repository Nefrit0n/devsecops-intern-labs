# webgoat-labs

Практикум по DevSecOps-сканированию на базе **OWASP WebGoat** как внешнего учебного полигона.

> Исходный код WebGoat не хранится в этом репозитории. Вы клонируете его отдельно из официального источника.

## 0) Подготовка и клонирование WebGoat

### Цель
Развернуть внешнее учебное приложение для выполнения security-лабораторных.

### Шаги
1. Перейдите в удобную директорию рядом с этим репозиторием.
2. Клонируйте официальный репозиторий WebGoat.
3. Запустите WebGoat (через Docker или локальную сборку).

### Команды
```bash
git clone https://github.com/WebGoat/WebGoat.git
cd WebGoat

# Вариант 1: Docker (пример)
docker build -t webgoat-local .
docker run --rm -p 8080:8080 -p 9090:9090 webgoat-local

# Вариант 2: локально (если нужен Maven/JDK)
# ./mvnw clean install
# java -jar webgoat-server/target/webgoat-server-*.jar
```

### Ожидаемый результат
- Приложение доступно локально (обычно `http://localhost:8080/WebGoat`).

### Вопросы студенту
- Почему WebGoat подходит как учебный полигон?
- Чем отличается практика на toy-приложении и на более крупном проекте?

### Критерии проверки
- Репозиторий WebGoat успешно клонирован.
- Приложение запускается локально.

## Лабораторная 2: SCA через Trivy FS

### Цель
Найти известные уязвимости в зависимостях и файлах проекта.

### Шаги
1. Запустите `trivy fs` по директории WebGoat.
2. Сохраните отчёт в JSON.

### Команды
```bash
cd WebGoat
trivy fs --scanners vuln,secret,misconfig --format json -o trivy-fs-report.json .
```

### Ожидаемый результат
- Получен отчёт с найденными проблемами (если есть).

### Вопросы студенту
- Какие CVE критичнее для приоритета исправления?
- Почему версия зависимости важна для риска?

### Критерии проверки
- Есть отчёт `trivy-fs-report.json`.
- Есть краткий вывод по severity.

## Лабораторная 3: Secret scanning через Gitleaks

### Цель
Проверить проект на потенциальные утечки секретов.

### Шаги
1. Запустите gitleaks в режиме сканирования директории.
2. Сохраните отчёт.

### Команды
```bash
cd WebGoat
gitleaks detect --source . --report-path gitleaks-report.json --report-format json
```

### Ожидаемый результат
- Сформирован отчёт о потенциальных секретах.

### Вопросы студенту
- Как отличать реальный секрет от ложноположительного срабатывания?
- Какие секреты особенно критичны (token, private key, cloud creds)?

### Критерии проверки
- Есть `gitleaks-report.json` и комментарии по findings.

## Лабораторная 5: Container scanning через Trivy Image

### Цель
Проверить уязвимости в контейнерном образе.

### Шаги
1. Соберите Docker-образ WebGoat.
2. Запустите `trivy image`.

### Команды
```bash
cd WebGoat
docker build -t webgoat-local .
trivy image --format json -o trivy-image-report.json webgoat-local
```

### Ожидаемый результат
- Получен отчёт по OS и library vulnerabilities внутри образа.

### Вопросы студенту
- Почему важно сканировать и source, и image?
- Что даёт фиксирование base image по digest?

### Критерии проверки
- Есть `trivy-image-report.json`.
- Есть список топ-рисков и рекомендации.

## Лабораторная 6: Генерация SBOM через Syft

### Цель
Сформировать Software Bill of Materials для проекта/образа.

### Шаги
1. Запустите syft по директории или образу.
2. Сохраните SBOM в SPDX JSON.

### Команды
```bash
cd WebGoat
syft dir:. -o spdx-json=sbom-webgoat.spdx.json
# или
syft webgoat-local -o spdx-json=sbom-webgoat-image.spdx.json
```

### Ожидаемый результат
- Сформирован SBOM со списком компонентов.

### Вопросы студенту
- Как SBOM помогает в incident response?
- Почему SBOM важен для прозрачности цепочки поставки?

### Критерии проверки
- Есть файл SBOM и краткая интерпретация.
